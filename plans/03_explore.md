# 03. 탐색 + 시각화

**우선순위**: 낮음. 데이터가 어느 정도(최소 1~2주) 쌓이고 나서 진행.

목표는 "이슈 파악". 정밀한 분석이 아니라, **"지금 뭐가 움직이고 있나"**를 빨리 보는 것.

---

## T1. 확률 시계열 차트 (가장 단순)

- [ ] 마켓 ID/slug 지정 → 시간축 vs yes_price 라인 차트
- [ ] matplotlib 또는 plotly (interactive 원하면 plotly)
- [ ] 노트북 한 개에서 시작 (`explore/probability_chart.ipynb`)

→ verify: 임의 마켓 1개의 확률 추이가 차트로 출력됨

## T2. 급변 감지

- [ ] 단순 룰부터: "최근 24시간 동안 ±10%p 이상 변동"
- [ ] 임계값은 데이터 분포 보고 조정 (미리 고정 X)
- [ ] 출력: 급변 마켓 리스트 + 변동폭

→ verify: 1주일 데이터에서 급변 이벤트 후보 3개 이상 추출

## T3. 크로스 플랫폼 비교

- [ ] 동일 사건 (예: Fed 금리)의 Polymarket vs Kalshi 확률 오버레이
- [ ] 매핑 테이블은 [01_collectors.md](01_collectors.md) T4에서 만들어진 것 사용
- [ ] 두 플랫폼 간 괴리/시차 보이는지 눈으로 확인

→ verify: 동일 이벤트 1개 이상 오버레이 차트 출력

## T4. 통합 뷰 (선택)

- [ ] Streamlit 로컬 앱 한 페이지 — 마켓 리스트 + 클릭 시 차트
- [ ] 만들지 말지는 노트북이 충분한지 보고 결정

**경고**: 대시보드 만들기에 빠지면 분석은 미뤄짐. 진짜로 매일 보게 됐을 때만 만들 것.
