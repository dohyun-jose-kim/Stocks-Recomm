# 00. Overview

## 확정 사항
- **언어**: Python only
- **스코프**: 예측시장만 (Polymarket + Kalshi). 한국 뉴스/가격/매크로/이벤트 캘린더 전부 제외.
- **목표**: 군중이 어떤 이슈를 어떻게 보는지 따라가기. 재미 + 시계열 축적.
- **실행 환경**: macOS 로컬, SQLite, launchd

## 미결 사항
- [ ] 관심 카테고리/마켓 화이트리스트 — 처음에는 넓게 잡고 운영하면서 좁히기
- [ ] 스냅샷 주기 — 30분으로 시작 (수정 가능)
- [ ] Kalshi 마켓 분류 (KXFED, KXCPI 등) 최신 코드 확인

## 실행 순서

```
Step 1  수집기 MVP
        - Polymarket: 활성 이벤트 → outcomePrices 스냅샷
        - Kalshi: 관심 시리즈 → yes_bid/yes_ask 스냅샷
        - 30분 간격, SQLite에 저장

Step 2  운영 안정화
        - 24시간 무중단 운영 확인 (48 스냅샷 ±10%)
        - 에러 로그 정리, 재시도 동작 확인

Step 3  탐색 (데이터가 쌓이고 나서)
        - 확률 시계열 차트
        - 급변 감지 (임계값은 분포 보고 결정)
        - Polymarket vs Kalshi 동일 이벤트 비교
```

데이터가 쌓이지 않으면 분석할 게 없다. 그래서 **수집기를 먼저 돌리고, 그 사이에 탐색은 뒤로 미룬다.**

## 의도적으로 제외한 것

이전 계획에서 빼놓은 것 (`archive/2026-04-19_quant-scope/`에 보존):

- 한국 뉴스 수집 (네이버 API, RSS)
- 가격 데이터 (yfinance, pykrx)
- 매크로 지표 (FRED, ECOS)
- 이벤트 캘린더 (DART, FRED 릴리즈)
- 뉴스 센티먼트 (FinBERT)
- BERTopic 토픽 모델링
- 뉴스↔시장 교차 분석
- 통합 대시보드
- 퀀트 학습 로드맵 (CAPM, 파마-프렌치 등)

**왜 뺐나**: 퀀트급 시스템을 만들기엔 현실적으로 어렵다고 판단. 예측시장 한 축만 깊이 하기로 함.

**다시 살릴 가능성**: 예측시장 데이터를 보다가 "이 사건 한국 뉴스에서는 언제 보도됐지?" 같은 진짜 질문이 생기면 그때 부활 후보. 미리 만들지 않음.
