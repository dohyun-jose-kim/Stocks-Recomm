# 프로젝트 개요

## 한 줄 요약
사람이 다 볼 수 없는 시장 시그널을 자동으로 수집·분석하여, 매매 판단의 참고 지표로 제공하는 프로그램.

## 왜 하는가
매매할 때 봐야 할 게 너무 많다. 뉴스, 경제지표, 금리, 공시, 예측시장 확률, 센티먼트 — 이걸 사람이 실시간으로 전부 추적하는 건 불가능하다. 그래서 프로그램이 대신 읽고, 정리하고, 시그널을 뽑아준다.

구체적으로:
1. **뉴스 흐름을 수치화한다.** 어떤 토픽이 급등하는지, 시장 센티먼트가 어디로 기울고 있는지를 숫자로 보여준다.
2. **예측시장이 말하는 확률을 추적한다.** Polymarket, Kalshi에서 Fed 금리, CPI, 경기침체 확률 같은 군중 판단을 실시간으로 모은다. 이 데이터가 뉴스보다 빠른지, 시장보다 빠른지를 본다.
3. **흩어진 지표를 한 곳에 모은다.** 한국/미국 가격, 경제지표, 공시, 이벤트 캘린더까지 — 매매 판단에 필요한 맥락을 한 화면에서 볼 수 있게 한다.

## 무엇을 하는가
- **수집**: 한국 금융 뉴스, 미국 예측시장 확률, 한국/미국 주가, 경제지표, 이벤트 캘린더를 자동으로 모은다.
- **분석**: 뉴스 토픽·센티먼트 추출, 예측시장 확률 급변 감지, 뉴스↔시장 간 선후행 관계 탐색.
- **시그널 제공**: "지금 시장 심리가 어떤지", "예측시장이 뭘 말하는지", "뉴스 흐름이 어디로 가는지"를 매매 참고 지표로 정리한다.

## 무엇이 아닌가
- 자동매매 봇이 아니다. 사고파는 건 사람이 한다.
- "내일 삼성전자 +3%"를 맞추는 예측기가 아니다. 판단 재료를 제공하는 것이지, 판단을 대신하지 않는다.

## 누가 하는가
- 개인 프로젝트. BERTopic 토픽 모델링, 텍스트마이닝, Transformer, RAG 경험을 금융 도메인으로 확장.

## 기술 스택
- **언어**: Python
- **데이터 저장**: SQLite (→ 필요 시 PostgreSQL)
- **NLP**: BERTopic, FinBERT, ko-sbert-nli
- **데이터 소스**: 네이버 뉴스 API, RSS, Polymarket API, Kalshi API, yfinance, pykrx, FRED, ECOS, DART
- **실행 환경**: macOS 로컬, launchd 스케줄링

## 프로젝트 구조
```
plans/          ← 계획 문서 (세부 태스크 분할)
src/            ← 소스 코드 (구현 시 생성)
  collector/    ← 데이터 수집기
  processor/    ← 전처리
  analyzer/     ← 분석
  storage/      ← DB
  dashboard/    ← 시각화
  config/       ← 설정/워치리스트
research/       ← 선행연구, 질문 설계, 설계서
```

## 현재 상태
- **단계**: 계획 수립 + 수집기 구현 준비
- **확정 사항**: Python only, 한국+미국 스코프, 예측시장 수집 우선 시작
- **다음 단계**: 예측시장 수집기 MVP → 뉴스 수집기 → 나머지 데이터 소스
