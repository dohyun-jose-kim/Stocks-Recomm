# 08. 리서치 — 선행연구 / 질문 설계

**우선순위**: 높음 (수집과 병렬로 진행)

---

## 태스크

### T8.1 선행연구 조사 (Week 1)
- [ ] 검색 키워드:
  - "news sentiment Korean stock market"
  - "topic modeling financial news"
  - "prediction market information efficiency"
  - "BERTopic finance"
  - "news lead-lag stock market"
- [ ] Google Scholar, arXiv (q-fin, cs.CL), SSRN
- [ ] 핵심 논문 5~10편 스키밍 (논문당 15~20분)
- [ ] 메모 형식:
  ```
  논문: [제목]
  질문: [한 줄]
  데이터: [소스, 기간, 규모]
  방법: [모델/분석]
  결과: [핵심 수치]
  한계: [저자가 인정한 것]
  내 프로젝트와 연결: [어떻게 참고/차별화?]
  ```
  - → verify: 5편 이상 메모 작성 완료

### T8.2 질문 후보 작성 (Week 1 후반)
- [ ] 축 1 (뉴스) 질문 후보 3~5개
- [ ] 축 2 (예측시장) 질문 후보 3~5개
- [ ] 각 질문에: 필요 데이터, 방법론, 예상 난이도 메모
  - → verify: 축별 3개 이상 후보, 각각 데이터/방법론 메모 포함

### 질문을 만드는 공식 (참고)
```
[데이터 A]의 [특성 X]가 [데이터 B]의 [특성 Y]에 [관계 Z]가 있는가?
조건: [시간 범위], [대상 범위], [통제 변수]
```

### T8.3 질문 평가 + 확정 (Week 2)
- [ ] 5가지 기준으로 점수:

  | 기준 | 설명 |
  |------|------|
  | 구체적 | 변수, 대상, 시간 범위가 명확 |
  | 검증 가능 | Yes/No 또는 수치로 답할 수 있음 |
  | 데이터 접근 가능 | 필요한 데이터를 실제로 구할 수 있음 |
  | 스코프 한정 | 한 사람이 합리적 기간 내 완료 가능 |
  | 실패해도 가치 | "상관 없음"도 의미 있는 결론 |

- [ ] 축 1에서 1개, 축 2에서 1개 선정
- [ ] 선정 이유 문서화
  - → verify: 확정 질문 2개 + 각각 5가지 기준 점수 존재

### T8.4 방법론 설계 (Week 2)
- [ ] 확정 질문별: 데이터 범위, 전처리, 분석 방법, 평가 지표
- [ ] 성공/실패 기준 정의
- [ ] 산출물 형태 확정 (리포트? 노트북? 대시보드?)
  - → verify: 질문당 "이 수치가 이 범위면 유의미" 기준 명시

### T8.5 프로젝트 설계서 v1 (Week 2 마감)
- [ ] 질문 + 데이터 + 방법론 + 일정 + 평가 기준 통합 문서
  - → verify: 설계서 읽고 바로 구현 시작할 수 있는 상태

### 흔한 함정 (참고)
1. **도구부터 고르는 함정**: "BERTopic 쓸 거니까 토픽 모델링 하자" — 도구가 질문을 정하면 안 됨
2. **스코프 확장 함정**: 하고 싶은 게 늘어남 → 2주 후에는 반드시 1개로 수렴
3. **완벽한 질문 함정**: "충분히 구체적이고 답할 수 있는" 수준이면 시작해도 됨
4. **선행연구 무시 함정**: 이미 실패로 증명된 길을 반복하게 됨
