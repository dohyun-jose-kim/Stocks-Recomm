# 2주 플래닝 가이드: 리서치 질문 설계부터 프로젝트 확정까지

## 이 문서의 목적

지금까지 만든 문서들은 "무엇을 할 수 있는가"에 대한 조사다.
이 문서는 **"무엇을 할 것인가"를 결정하는 2주의 과정**을 설계한다.

2주 후 목표 산출물: **프로젝트 설계서 1장** — 질문, 데이터, 방법론, 평가 기준이 확정된 상태.

---

## 좋은 리서치 질문의 조건

질문이 없으면 수집기만 만들다 끝난다. 질문이 프로젝트의 뼈대다.

좋은 질문은 아래 5가지를 만족한다:

| 조건 | 설명 | 나쁜 예 | 좋은 예 |
|------|------|---------|---------|
| **구체적** | 변수, 대상, 시간 범위가 명확 | "뉴스로 주가 예측" | "한국 금융 뉴스 토픽 비중 변화가 KOSPI 섹터 수익률에 1~5일 선행하는가?" |
| **검증 가능** | Yes/No 또는 수치로 답할 수 있음 | "뉴스가 시장에 영향을 줄까?" | "토픽 X 비중이 상위 10%일 때, 해당 섹터 다음 주 수익률이 유의미하게 다른가?" |
| **데이터 접근 가능** | 필요한 데이터를 실제로 구할 수 있음 | "HFT 오더플로우 분석" (데이터 없음) | "네이버 뉴스 API + KOSPI 일봉" (둘 다 무료) |
| **스코프가 한정적** | 한 사람이 합리적 기간 내 완료 가능 | "글로벌 뉴스 → 전 세계 주가 예측" | "한국 뉴스 → KOSPI 5개 섹터, 최근 3년" |
| **실패해도 배울 게 있음** | "상관 없음"도 의미 있는 결론 | (해당 없음) | "선행성이 없다면, 뉴스는 시장을 반영하지 예측하지 못한다는 증거" |

### 질문을 만드는 공식

```
[데이터 A]의 [특성 X]가 [데이터 B]의 [특성 Y]에 [관계 Z]가 있는가?

조건: [시간 범위], [대상 범위], [통제 변수]
```

예시:
- [한국 금융 뉴스]의 [BERTopic 토픽 비중 변화]가 [KOSPI 섹터 ETF]의 [주간 수익률]에 [1~5일 선행 상관]이 있는가?
- [Polymarket 내재확률 변동]이 [관련 한국 뉴스 보도량 증가]에 [시간적으로 선행]하는가?

## 축 1 (뉴스 기반 분석) — 질문 후보 탐색 방향

본인 강점: BERTopic, 텍스트마이닝. 이걸 살리는 방향도 좋음
이거 국한 되지 말 것

### 탐색할 질문 영역들

**A. 토픽 → 시장 반응 (가장 자연스러운 확장)**
- 뉴스 토픽 분포의 변화가 섹터 수익률/변동성에 선행하는가?
- 새로운 토픽의 등장(emergence) 속도와 시장 반응 크기 간 관계?
- 토픽 집중도(소수 토픽 과점)가 시장 변동성의 선행지표가 되는가?

**B. 센티먼트 극단값 → 시장 반응**
- FinBERT 센티먼트 극단값(극도로 부정적 기사 폭증)이 단기 하락에 선행하는가?
- 센티먼트와 토픽을 결합하면 단독보다 예측력이 올라가는가?

**C. 정보 전파 구조**
- 특정 매체가 다른 매체보다 시장 반응에 선행하는가? (예: 한경 vs 연합 vs 네이버 블로그)
- 뉴스 간 토픽 전파 패턴이 존재하는가?

### 선행연구에서 확인할 것
- 한국 시장 대상 뉴스-주가 연구가 있는가? 어떤 데이터, 어떤 결과?
- BERTopic을 금융 뉴스에 적용한 사례? (대부분 LDA 기반일 것)
- 토픽 모델링 기반 "예측"이 아닌 "분석"으로 포지셔닝한 연구?

---

## 축 2 (예측시장 수집) — 질문 후보 탐색 방향

데이터가 지금부터 쌓이므로, 질문은 **"지금 설계해두고 나중에 답하는"** 형태.

### 탐색할 질문 영역들

**A. 정보 시차**
- 예측시장 확률 급변이 뉴스 보도보다 먼저 움직이는가?
- 아니면 뉴스가 먼저이고 예측시장이 따라가는가?

**B. 플랫폼 간 비교**
- Polymarket과 Kalshi의 동일 이벤트 확률 괴리는 수렴하는가?
- 괴리 크기가 이벤트 유형(경제지표 vs 정치)에 따라 다른가?

**C. 예측시장 → 한국 시장 연결 (축 1과 합류점)**
- Fed 금리 결정 Polymarket 확률 변동이 한국 뉴스 토픽 변화에 선행하는가?
- 예측시장 확률이 한국 시장 개장 시 갭에 반영되는가?

### 축 2의 핵심 제약
- 과거 데이터 없음 → 백테스팅 불가
- 의미 있는 분석에 최소 1~2개월 데이터 필요
- 따라서 2주 플래닝 기간에는 **수집 설계 + 질문 확정**까지만. 분석은 미래.

## 2주 플래닝 일정

### 1주차: 탐색과 발산

| 일 | 할 일 | 산출물 |
|----|-------|--------|
| Day 1~2 | **선행연구 조사** — Google Scholar, arXiv에서 "news sentiment Korean stock market", "topic modeling financial news", "prediction market information" 검색. 핵심 논문 5~10편 스키밍. | 논문 요약 리스트: 각 논문의 질문, 데이터, 결과, 한계 (1줄씩) |
| Day 3 | **데이터 접근 테스트** — 네이버 뉴스 API 실제 호출. 어떤 데이터가 오는지, 과거 데이터 범위는 어디까지인지 확인. Polymarket/Kalshi API도 curl 테스트. | API 테스트 결과 메모: 실제 응답 예시, 제약 사항 |
| Day 4~5 | **질문 후보 작성** — 위 탐색 결과를 바탕으로 축 1, 축 2 각각 질문 후보 3~5개 작성. 각 질문에 대해: 필요 데이터, 가능한 방법론, 예상 난이도 메모. | 질문 후보 리스트 (축별 3~5개) |

### 2주차: 수렴과 확정

| 일 | 할 일 | 산출물 |
|----|-------|--------|
| Day 6~7 | **질문 평가** — 후보 질문들을 위의 5가지 조건(구체적, 검증 가능, 데이터 접근 가능, 스코프 한정, 실패해도 가치)으로 점수 매기기. 축 1에서 1개, 축 2에서 1개 선정. | 확정 질문 2개 + 선정 이유 |
| Day 8~9 | **방법론 설계** — 확정 질문에 대해: 데이터 수집 범위, 전처리 방법, 분석 방법, 평가 지표 구체화. 백테스팅 설계(축 1). 수집 스키마 확정(축 2). | 방법론 초안 |
| Day 10 | **성공/실패 기준 정의** — "이 수치가 이 범위면 유의미", "이 결과면 귀무가설 기각 불가" 등. 산출물 형태 확정 (리포트? 노트북? 대시보드?). | 평가 기준 문서 |
| Day 11~12 | **최종 설계서 작성** — 질문, 데이터, 방법론, 일정, 평가 기준을 하나의 문서로 통합. 구현 시작 전 최종 점검. | **프로젝트 설계서 v1** |

### 각 단계의 체크포인트

```
□ 1주차 끝: "내가 답하려는 질문 후보가 있고, 각각에 필요한 데이터를 확인했다"
□ 2주차 끝: "질문 1개가 확정되었고, 어떤 데이터를 어떤 방법으로 분석해서
             어떤 기준으로 평가할지 문서화되었다"
```

## 선행연구 조사 가이드

2주 중 가장 중요한 단계. 방법:

### 어디서 찾나
- **Google Scholar**: "news sentiment stock prediction", "topic modeling financial markets"
- **arXiv (q-fin, cs.CL)**: 최신 프리프린트
- **SSRN**: 금융 실무 리서치
- **GitHub**: 관련 repo의 README에 인용된 논문들

### 어떻게 읽나 (스키밍)
논문 전체를 읽지 않는다. 논문당 15~20분:
1. **Abstract** — 질문과 결론
2. **Data section** — 어떤 데이터를 썼는지 (내가 구할 수 있는 건지?)
3. **Results table** — 실제 성능/상관 수치
4. **Conclusion의 limitations** — 내가 다르게 할 수 있는 부분?

### 메모 형식
```
논문: [제목]
질문: [한 줄]
데이터: [소스, 기간, 규모]
방법: [모델/분석]
결과: [핵심 수치]
한계: [저자가 인정한 것]
내 프로젝트와 연결: [어떻게 참고/차별화?]
```

## 흔한 함정

1. **도구부터 고르는 함정**: "BERTopic 쓸 거니까 토픽 모델링 하자" — 도구가 질문을 정하면 안 된다. 질문이 먼저, 도구는 그다음.
2. **스코프 확장 함정**: 조사하다 보면 하고 싶은 게 늘어남. "이것도 재밌겠다"를 전부 넣으면 아무것도 못 끝냄. **2주 후에는 반드시 1개로 수렴.**
3. **완벽한 질문 함정**: 완벽한 질문은 없다. "충분히 구체적이고 답할 수 있는" 수준이면 시작해도 된다. 구현하면서 질문이 다듬어지는 건 자연스러운 일.
4. **선행연구 무시 함정**: "그냥 내 방식대로 하겠다"하면 이미 실패로 증명된 길을 반복하거나, 이미 답이 나온 질문을 다시 풀게 됨.
